Building a Raspberry Pi 5 Homelab: DNS, NAS, and Observability

Turning a small board into a full lab for networking, storage, and monitoring.
Raspberry Pi 5 AdGuard NextDNS NAS Grafana

A Raspberry Pi 5 is more than fast enough to act as a small homelab if you design it carefully. In this write-up, I walk through how I wired mine to handle DNS filtering, local NAS storage, and observability for my home network.

High-Level Architecture

The Pi connects to the router via Ethernet and runs a small stack of services:

  • AdGuard Home in front of NextDNS for DNS filtering.
  • Samba to expose a 2 TB drive as a simple NAS.
  • Node exporter + Prometheus + Grafana for metrics.

DNS: AdGuard + NextDNS

AdGuard runs locally and forwards upstream to NextDNS, which lets me keep profile-based blocking while still having a single, fast local IP for all devices. The router’s DNS is set to the Pi’s address, and AdGuard handles the rest.

NAS on a Budget

Instead of buying a dedicated NAS, I attached a USB 3 drive and exported it using Samba. It’s not Synology, but for photo backups and lab files it works great.

Observability

Exporters on the Pi feed metrics into Prometheus. Grafana dashboards show:

  • CPU and memory usage on the Pi.
  • Network throughput to and from the router.
  • DNS query volumes and block rates from AdGuard.

The result is a tiny, always-on lab that keeps my network faster and safer, and gives me a place to test ideas without touching production AWS accounts.
